SESSION 003 — F004 (foundation)  ·  2026-07-17  ·  model: claude-opus-4-8
-------------------------------------------------------------------------------
FEATURE: F004 — the dockerized stack serves the app end-to-end (frontend +
backend + database, all from one docker-compose environment). Phase 0 of the
demo-priority track.

WORK (real fix, not just verify):
- Startup verification exposed a latent stack bug: `docker compose ps` showed
  backend + frontend as **(unhealthy)** even though init.sh/endpoints worked.
  Root cause: both healthchecks probed `http://localhost:<port>` from inside the
  container; `localhost` resolves to IPv6 `::1` first, but the Node servers listen
  on IPv4 `0.0.0.0` only → wget got "Connection refused" every time.
- Fix: point both healthchecks at `127.0.0.1` (docker-compose.yml backend :3000
  and frontend :4200). Recreated containers → all three services now report
  **healthy**, making this a genuinely healthy end-to-end dockerized stack.

VERIFIED (browser, as a human):
